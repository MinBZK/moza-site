  <section class="show-controls hide-logo">
    <h3>Take-aways van deze sessie</h3>
    <ul>
      <li>
        Wisselend draagvlak voor alleen profiel service. Wel draagvlak voor de combinatie van profiel en
        notificatie.
      </li>
      <li>Er zou een 'Diensten Catalogus' moeten komen voor dienst-specifieke voorkeuren</li>
      <li>eHerkenning blijft een puzzel, verdere verkenning nodig</li>
      <li>Eenvoudiger samenwerken aan de documentatie; bijv. aansluiten op Notion van VNG</li>
    </ul>
  </section>
  <section class="show-controls hide-logo">
    <h3>Take-aways van deze sessie</h3>
    <ul>
      <li>
        We willen van dienstverleners weten op basis van welke identificatienummers er momenteel
        gegevenskoppelingen gemaakt worden
      </li>
      <li>Wat als een organisatie opmerkt dat een e-mailadres niet langer meer geldig is?</li>
      <li>
        Toch gegevens uit externe bronnen (KVK Handelsregister, BRP, etc) koppelen ten behoeve van gebruiksgemak
        afnemers
        <ul>
          <li>Naam bij contactvoorkeur bewaren voor gebruik als "ter attentie van"</li>
        </ul>
      </li>
    </ul>

    <h3>Federatief Datastelsel (FDS)</h3>
    <img src="/content/moza_pulse_20251202/fds-standaard.jpg" />
  </section>
  <section class="show-controls hide-logo">
    <h3>Federatieve Service Connectiviteit (FSC)</h3>
    <p>
      Standaard die zorgt voor veilige gegevensuitwisseling tussen (overheids)diensten. Zorgt dat:
      <ul>
        <li>koppelingen standaard en flexibel zijn,</li>
        <li>combineert met Federatieve ToegangsVerlening (FTV), en</li>
        <li>Logboek Dataverwerking (LDV).</li>
      </ul>
    </p>
    Zie:
    <a href="https://fsc-standaard.nl/">https://fsc-standaard.nl/</a>
  </section>
  <section class="show-controls hide-logo">
    <h3>Federatieve Service Connectiviteit (FSC)</h3>
    <img src="/content/moza_pulse_20251202/fsc-standaard.png" />
    - https://simulatie.datastelsel.nl/environments
  </section>
  <section class="show-controls">
    <h3>Federatieve ToegangsVerlening (FTV)</h3>
    <p>
      Zorgt dat toegangsverlening via Policy Based Acces Control (PBAC) onder de juiste voorwaarden gebeurt,
      werkt samen met FSC en LDV.
    </p>
    Zie:
    <a href="https://vng-realisatie.github.io/ftv/">https://vng-realisatie.github.io/ftv/</a>
  </section>
  <section class="show-controls">
    <h3>Logboek DataVerwerkingen (LDV)</h3>
    <p>
      Zorgt dat gegevensverwerkingen eenduidig worden vastgelegd en onderling - ook over organisaties heen -
      relateerbaar zijn. Doordat iedereen deze standaard gebruikt, zijn logs te combineren en kan het
      gegevengebruik door de keten verantwoord worden.
    </p>
    Zie:
    <a href="https://logius-standaarden.github.io/logboek-dataverwerkingen/"
      >https://logius-standaarden.github.io/logboek-dataverwerkingen/</a
    >
  </section>
</section>

    <h3>LDV standaard geïmplementeerd in de Profiel Service</h3>
    <ul>
      <li>Feedback op LDV standaard kunnen leveren, waarmee de standaard wordt aangescherpt</li>
      <li>
        Java implementatie als Open Source beschikbaar op:
        <a href="https://github.com/MinBZK/moza-logboekdataverwerking"
          >https://github.com/MinBZK/moza-logboekdataverwerking</a
        >
      </li>
      <li>Opgenomen als referentie implementatie op <a href="https://developer.overheid.nl/kennisbank/data/standaarden/logboek-dataverwerkingen/implementaties/jakarta">developer.overheid.nl</a>.</li>
    </ul>
  </section>
  <section class="show-controls">
    <h3>Profiel Service via FSC beschikbaar</h3>
    <ul>
      <li>
        Profiel Service is via het FSC als register beschikbaar gesteld binnen de
        <a href="https://simulatie.datastelsel.nl/environments">FDS simulatie omgeving</a>.
      </li>
      <li>
        Het FSC zorgt voor betrouwbare en veilige ontsluiting van de Profiel Service voor overheidsorganisaties.
      </li>
      <li>Hiermee kunnen we pilots draaien met afnemende organisaties.</li>
    </ul>
  </section>
